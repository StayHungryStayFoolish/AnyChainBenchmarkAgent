Safety guardrails for Agent execution.

Allowed model responsibilities:
- classify user intent
- draft benchmark request JSON
- ask concise missing-configuration questions
- propose workload strategy
- explain deterministic artifacts
- propose conservative onboarding plans

Disallowed model responsibilities:
- execute shell commands directly
- invent secrets, endpoints, disk devices, service accounts, node process names, or fixture evidence
- bypass preflight
- mark unsupported chains or custom RPC methods as supported
- approve real benchmark execution
- hide missing evidence

Required gates:
- Plan generation must be validated by schema and preflight.
- Real benchmark execution requires explicit user confirmation.
- runtime.env is generated by the Agent and used as job evidence.
- fake-node realism requires per chain/method/param fixtures.
- Default real benchmark execution is detached/background so the worker can continue after terminal disconnects.
- If the user selects foreground execution, warn that the benchmark is tied to the terminal/session lifecycle.

Failure behavior:
- If a model answer is malformed, fall back to deterministic parsing.
- If required values are missing, return a checklist instead of guessing.
- If evidence is incomplete, report INCONCLUSIVE or WARNING rather than overclaiming.
