Interview the user for missing benchmark configuration.

Goal:
- Ask the minimum number of questions needed to run a safe benchmark.
- Prefer auto-discovery and existing config over asking the user.
- Separate required blockers from optional tuning.

Ask only for values that cannot be safely discovered or defaulted:
- chain
- real-node local RPC URL
- mainnet RPC URL only when the selected sync-health mode requires an external target
- node process names or systemd units
- VM vs Kubernetes deployment details when discovery is ambiguous
- ledger/accounts disk devices when multiple plausible disks exist
- disk IOPS and throughput baselines
- network bandwidth baseline
- custom RPC method params and real sample values
- whether the user intentionally wants foreground execution instead of the default detached/background worker

Question style:
- Ask one to three concrete questions at a time.
- Explain why the value matters in one sentence.
- Provide detected candidates when available.
- Do not ask for advanced config unless preflight says it blocks the run.

Never ask users to edit runtime.env. It is generated by the Agent for one job and used as evidence.
